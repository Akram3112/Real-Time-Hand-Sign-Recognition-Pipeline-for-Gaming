# Naruto Hand Sign Recognition
> A real-time hand gesture recognition pipeline for Naruto-style *mudra* classification, built for the ELEN0449 Computer Vision Understanding course at ULiège.

**Course:** [ELEN0449 — Computer Vision Understanding](https://www.programmes.uliege.be/cocoon/20252026/cours/ELEN0449-1.html)  
**Instructor:** Anthony Cioppa  
**University:** University of Liège

---

## Overview

This project implements an end-to-end pipeline that recognizes the 12 Naruto hand signs (*mudra*) in real time using a standard webcam. Rather than processing raw images, the pipeline extracts compact 3D hand landmark features using **MediaPipe Hands** and feeds them into a lightweight classifier, reducing the input from hundreds of thousands of pixels to **126 features**.

The 12 classes correspond to the traditional zodiac-based hand signs from the Naruto universe: **Rat, Ox, Tiger, Hare, Dragon, Snake, Horse, Ram, Monkey, Bird, Dog, Boar**.

## Dataset

Videos were recorded by the authors and are **not publicly available**, as they were collected solely for research purposes within this course project. However, the extracted feature dataset (`4_dataset_POINT/`) is available and can be used directly to train new models or extend the dataset.

| Split      | Participants          | Frames |
|------------|-----------------------|--------|
| Train      | ~ 70%                 | 39,258 |
| Validation | 15%                   | 11,590 |
| Test       | 15%                   | 11,556 |


The split is **participant-wise** to prevent data leakage between sets. Each class contains approximately the same number of frames — no class imbalance.

Notable finding: some gestures (e.g. Dragon, Horse, Ox) exhibit high rates of missing landmarks on one hand due to occlusion when the two hands are in close proximity. See `5_feature_data_info/` for the full analysis.

## Getting Started

### Requirements

```bash
pip install mediapipe opencv-python numpy pandas scikit-learn matplotlib joblib torch 
```

### 1. Extract features from videos

```bash
python src/data_handling/1_feature_extractor_with_mediapipe.py
```

### 2. Preprocess the dataset

```bash
python src/data_handling/2_preprocess_features.py
```
### 3. Build train/val/test splits

```bash
python src/data_handling/3_build_dataset.py
```

### 4. Train the classifier

```bash
python src/training/1_train_with_dt.py
```

### 5. Run the live demo

```bash
python src/demo/1_demo_with_dt.py
```
```bash
python src/demo/2_demo_with_mlp.py
```
Press `q` to quit the demo.

To test the final demo of the project run:

```bash
python src/demo/3_final_demo.py
```
1-4 or F/L/W/S: choose jutsu
M: use MLP
D: use Decision Tree
SPACE/ENTER: start sequence
R: reset
Q/ESC: quit

## Notes

- MediaPipe is run in **VIDEO mode** with `num_hands=2`
- The first 2 seconds of each recording are discarded to avoid transition frames
- The final demo records predictions over a short time window, filters low-confidence predictions, and selects the final sign based on prediction count and average confidence.
- Mirror augmentation is applied during preprocessing.
- the missing landmark pattern is considered an inherent property of the gestures themselves and is preserved in the training data
